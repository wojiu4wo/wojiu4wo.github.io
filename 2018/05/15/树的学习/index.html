树的学习

Posté le 2018-05-15 | Visitors:

本文用于自己对于树的学习进行的一些总结，其中包括了各类帖子中的概念以及自己的一些思考。由于比较零碎，所以对于reference就不一一列出了。

二叉树(binary tree)

单纯的就是子节点的数量不超过2的树。因此每一个node最多只有左右子节点，且左右子节点的顺序不能改变。

分类

  • 满二叉树(full binary tree)： 除了external node, 每个internal node都有两个子节点
  • 完全二叉树(complete binary tree)： 出了最后一层节点，其余所有节点都是full binary tree。并且最后一层从左到右填满，没有空出来的节点。
  • 完美二叉树(perfect binary tree): external节点都在同一层，且所有internal节点都有左右子节点。

数据结构

  • array： 从root开始往下每一层从左到右遍历，
  • linked list： node.left 以及 node.right

二叉查找树 (binary search tree)

特性

  • 对于某节点的key来说，左节点的key比之小，右节点的key比之大
  • 对于某节点的key来说，左子树的所有节点都比之小，右子树的所有节点都比之大。
  • 任意节点的左右子树均为binary search tree

遍历

  • 前序遍历(preorder traversal): 访问root，前序遍历左子树，前序遍历右子树
  • 中序遍历(inorder traversal): 中序遍历左子树，访问根节点，中序遍历右子树
  • 后序遍历(postorder traversal): 后序遍历左子树，后序遍历右子树，访问根节点

优劣

  • 优：查找所需最大次数为树的高度
  • 劣势：会出现不平衡

BST的两儿子对比（红黑vsAVL）

这两者的算法研究有很多，因此仅仅给出一下结论即可。

  1. 红黑树没有严格限制节点高度之差为1，而仅仅得到平均高度为log n，最坏情况不超过 2log n。因此红黑是一个不严格的平衡树，而其不严格使得插入效率更高。
  2. avl更加的平衡，因此查找效率更高
  3. 旋转的时间复杂度为o(1)，因此两树的插入worst case都是o(log n)，但是统计上来说，红黑的插入和删除更好。
  4. AVL是最早的平衡树，不过当前红黑用的比较多。c++中STL的map和set是用红黑实现的。

AVL （平衡二叉树）

##概念

  • AVL来自于binary search tree， 所以拥有其特性。
  • 任意节点的左右子节点高度差绝对值不超过1.这个1也就是平衡因子。对于AVL的判断，可以从bottom-up方法不断的判断。
  • 注意子节点的高度有其最长路径决定，而不是最短路径

操作

插入

  • 从插入节点x开始，找到第一个失去平衡的节点，一般为祖父节点z，然后中间经过了父节点y。
  • 判断类型，从z开始出发，判断y的位置，x的位置，即为类型。
  • LL与RR单次rotation即可，LR和RL需要double rotation。
  • rotation的方法尽量以画图为准，基本idea就是y篡位成为root，z与x其子节点，原本y的子节点成为现在z的子节点。
  • 每一次rotation的时间复杂度为o(1)，每次rotation完都有可能造成新的不平衡，所以往往插入后会经历多次rotation直到到达root，因此worst-case与avg为o(log n)， h = log n

删除

删除某个节点后，同样产生不平衡，进行rotation即可，因此依然为o(log n)

搜索

与binary search tree 一样，搜索需要o(log n)。

搭建

按照binary search tree搭建方法进行搭建，但每加入一个新node都要进行一次平衡检测。

红黑树

概念

  1. 为解决二叉查找树的不平衡性而产生的，本质依然是二叉查找树，甚至是具有AVL特性的带颜色的树
  • 节点为红色或者黑色，root为黑色，external均为黑色空节点（NIL）
  • 每个红色节点的两个子节点为黑色
  • 任一节点到其每个external node的所有路径包含相同数量的黑色节点

以上特点保证了红黑树中root到externla node的最长路径不会超过最短路径的两倍。

搭建以及插入

step 1

从root以及两个NIL节点开始，对于红黑树来说，强行在原树的所有external node后再加入NIL节点。如果是根节点，染黑；如果不是根节点，先染红。

step 2

将现有树当作BST，将节点插入，染成红色。

step 3

第一种情况： 新插入节点为root，则直接染成黑色

第二种情况：新插入节点的父节点为黑色，不做变化，依然为红黑。

第三种情况：三种情况核心思想均为将红色节点移到根节点，然后将根节点变黑。

  • 新插入节点的父节点为红色，叔叔节点为红色: 将父节点变黑，叔叔节点变黑，祖父节点变红，将祖父节点设为当前节点，再次判断情况。
  • 新插入节点的父节点为红色，叔叔节点为黑色，且新插入节点为其父节点的右节点： 将父节点设为当前节点，对新的当前节点进行左旋
  • 新插入节点的父节点为红色，叔叔节点为黑色，且新插入节点为其父节点的左节点：将父节点变黑，祖父节点变红，以祖父节点进行右旋

具体操作查看红黑原理

删除

step 1

将红黑树当作普通BST进行删除：

  • 该节点为external node，则直接删除
  • 该节点只有一个子节点，则删除后，将其子节点顶替原来的位置
  • 该节点有两个子节点，找到后续节点（根据前序中序或者后序，一般为中序，因此为右孩子），后续节点顶替原来位置

step 2

通过rotation和变色来修正，x为y的后续节点，当y被删除以后，x顶替了y的位置，那么我们假设x拥有了两个颜色：x原本的颜色 以及 y的颜色。

第一种情况：x颜色为“红+黑”，则设x为黑
第二种情况：x颜色为”黑+黑”，且为root，则不做变化
第三种轻卡滚：x颜色为“黑+黑”，不是root，则

  • x的兄弟节点为红： x的兄弟节点变黑，x的父节点变红，x的父节点左旋，再重新判断
  • x的兄弟节点为黑，且该兄弟节点的两个孩子为黑： x的兄弟节点变红，x的父节点为新的当前节点，再次判断
  • x的兄弟节点为黑，且该兄弟节点左孩子为红，右孩子为黑： x的兄弟节点左孩子变黑，x的兄弟节点变红，x的兄弟节点右旋，重新判断

  • x的兄弟节点为黑，且该兄弟节点左孩子为任意颜色，右孩子为红： x的父节点颜色给x的兄弟节点，x的父节点变黑，x的兄弟节点右孩子变黑，x的父节点左旋，设x为当前节点
